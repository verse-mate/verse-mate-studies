# @versemate/studies

Versemate inductive Bible study content. Hendricks Observation–Interpretation–Application (OIA) method, NASB 1995, English-transliterated Greek with definition column, meaning-first interpretation movements.

Shared by **verse-mate-web** (Vite SPA) and **verse-mate-mobile** (Expo React Native).

## Usage

```ts
import { getStudyFor, type InductiveStudy } from '@versemate/studies';

const study = getStudyFor(62, 1); // 1 John 1
if (study) {
  // study.steps[0..8], study.interpretation, study.application
}
```

## Coverage

Generated content (NASB 1995, AI-generated via the verse-mate-web `scripts/study-ingest/` pipeline, with the 1 John 1 + James 1 hand-authored studies as one-shot quality targets):

- James 1-5
- 1 John 1-5
- NT batch: 250 chapters (in flight as of 2026-05-16)
- OT batch: TBD (929 chapters, requires Hebrew lexicon alignment first)

## Authoring pipeline

Content is generated by `verse-mate-web/scripts/study-ingest/generate.py` (single chapter) and `batch_generate.py` (bulk via Anthropic Batch API). The pipeline writes `<book-slug>-<chapter>.ts` files directly into `verse-mate-studies/src/`, then `rebuild_index.py` regenerates `src/index.ts`.

## Why a separate package?

The web app currently bundles studies as TypeScript modules at build time. When the React Native app adds a Study UI, it consumes the same package via Metro's native TS support — same content, same shape, zero duplication. If bundle size becomes a concern at full Bible coverage (~25 MB raw / ~5-7 MB compressed for 1,189 chapters), we migrate to API-served JSON — at which point this package's JSON output stays canonical.
